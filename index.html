Introducción a Bash Script

Fundamentos del shell scripting en sistemas Unix/Linux. Aprende a automatizar tareas y dominar la línea de comandos.

¿Qué es Bash?

Bash (Bourne Again SHell) es el intérprete de comandos más utilizado en sistemas Linux y macOS. Un script de Bash es simplemente un archivo de texto con una secuencia de comandos que el shell ejecuta de forma automatizada.

¿Para qué sirve?

Automatización de tareas, administración del sistema, procesamiento de archivos, despliegue de aplicaciones y muchas más.

Extensión del archivo

Los scripts Bash se guardan con extensión .sh aunque no es obligatorio — lo que importa es el shebang.

Dónde se usa

Linux, macOS, WSL (Windows Subsystem for Linux), servidores, CI/CD pipelines, Docker, etc.

Tu primer script

Todo script comienza con el shebang (#!/bin/bash), que indica al sistema operativo qué intérprete usar para ejecutar el archivo.

bash · hola_mundo.sh 
#!/bin/bash
# Este es un comentario — el intérprete lo ignora
# Siempre empieza con el shebang en la línea 1

echo "¡Hola, Mundo!"
echo "Hoy es: " $(date)
echo "Usuario: $USER"
echo "Directorio actual: $PWD"
Cómo ejecutarlo: Primero dale permisos de ejecución con chmod +x hola_mundo.sh, luego ejecútalo con ./hola_mundo.sh

Ejemplos prácticos de los comandos nuevos

bash · comandos_practicos.sh 
#!/bin/bash

## touch — crear archivo vacío o actualizar fecha
touch log_$(date +%Y%m%d).txt       # crea log_20240101.txt
touch -t 202301010000 archivo.txt  # fija fecha manualmente

## cp — copiar archivos
cp config.txt config.bak           # copia simple
cp -r carpeta/ carpeta_backup/    # copia recursiva de directorio
cp -p original.sh copia.sh        # preserva permisos y fechas

## mv — mover o renombrar
mv script_v1.sh script_v2.sh      # renombrar
mv *.log logs/                    # mover todos los .log a carpeta logs/

## cut — extraer columnas
# -d define el delimitador, -f el número de campo
cut -d: -f1 /etc/passwd          # → nombres de usuarios
cut -d, -f2,4 datos.csv          # → columnas 2 y 4 del CSV
cut -c1-5 archivo.txt             # → primeros 5 caracteres de cada línea

## tr — traducir / eliminar caracteres
echo "hola mundo" | tr 'a-z' 'A-Z'  # → HOLA MUNDO
echo "a,b,c" | tr ',' ' '          # → a b c
echo "hola  mundo" | tr -s ' '    # → hola mundo (comprime espacios)
echo "abc123" | tr -d '0-9'       # → abc (elimina dígitos)

## wc — contar
wc -l archivo.txt                  # número de líneas
wc -w archivo.txt                  # número de palabras
wc -c archivo.txt                  # número de bytes
echo "hola" | wc -c               # → 5 (4 letras + \n)
total=$(wc -l < archivo.txt)       # guardar en variable (sin nombre de archivo)

## awk — procesamiento de campos
# Estructura: awk 'condición { acción }' archivo
awk '{print $1}' datos.txt         # primera columna de cada línea
awk '{print $1, $NF}' datos.txt    # primera y última columna
awk -F: '{print $1, $3}' /etc/passwd  # delimitador personalizado (:)
awk '{suma += $1} END {print suma}' nums.txt  # sumar una columna
awk 'NR==3' archivo.txt            # mostrar solo la línea 3
awk '$3 > 100' ventas.txt          # filtrar filas donde col3 > 100
df -h | awk 'NR>1 {print $1, $5}'  # disco: dispositivo y uso%

## set — opciones del shell
set -euo pipefail   # combinación recomendada para producción
set -x               # activar debug (muestra cada comando ejecutado)
x=5; y=10
set +x               # desactivar debug
set -- "arg1" "arg2"  # redefine los argumentos posicionales $1, $2...

Códigos de salida

Cada comando devuelve un código de salida. 0 significa éxito, cualquier otro número indica error. Se accede con la variable especial $?.

#!/bin/bash

Variables

Almacenar, acceder y manipular datos en Bash con variables locales, de entorno y especiales.

Declaración y uso

En Bash, las variables se crean simplemente asignando un valor. No hay espacios alrededor del =. Para acceder al valor, se usa el prefijo $.

bash · variables.sh 
#!/bin/bash

# Asignación — SIN espacios alrededor del =
nombre="Ana"
edad=25
pi=3.14159
activo=true

# Acceso con $
echo "Nombre: $nombre"
echo "Edad: $edad años"

# Sintaxis con llaves — recomendada para evitar ambigüedades
echo "Bienvenida, ${nombre}!"
echo "${nombre}_backup"   # Sin llaves interpretaría $nombre_backup

# Variable de solo lectura (constante)
readonly VERSION="1.0.0"
echo "Versión: $VERSION"

# Eliminar una variable
unset activo
Atención: nombre = "Ana" con espacios causará un error — Bash intentará ejecutar nombre como un comando.

Variables de entorno

Las variables de entorno son globales al sistema. Por convención se escriben en MAYÚSCULAS. Puedes exportar variables locales al entorno con export.

Tipos de datos y operadores

Cadenas de texto, números, aritmética y comparaciones en Bash.

Cadenas de texto

bash · cadenas.sh 
#!/bin/bash

saludo="Hola, Mundo"

# Longitud de cadena
echo "Longitud: ${#saludo}"            # → 11

# Subcadena: ${variable:inicio:longitud}
echo "${saludo:0:4}"                  # → Hola
echo "${saludo:6}"                   # → Mundo

# Reemplazar primera ocurrencia
echo "${saludo/Hola/Adiós}"         # → Adiós, Mundo

# Reemplazar todas las ocurrencias
texto="uno dos uno tres uno"
echo "${texto//uno/1}"              # → 1 dos 1 tres 1

# Mayúsculas / Minúsculas (Bash 4+)
nombre="juan garcia"
echo "${nombre^^}"                  # → JUAN GARCIA
echo "${nombre^}"                   # → Juan garcia (solo 1ª letra)
TEXTO="HOLA MUNDO"
echo "${TEXTO,,}"                  # → hola mundo

# Concatenar cadenas
a="Bash"
b="Script"
c="$a $b"
echo $c                            # → Bash Script

Aritmética

bash · aritmetica.sh 
#!/bin/bash

a=10
b=3

# Operadores básicos con $(( ))
echo $((a + b))   # Suma → 13
echo $((a - b))   # Resta → 7
echo $((a * b))   # Multiplicación → 30
echo $((a / b))   # División entera → 3
echo $((a % b))   # Módulo → 1
echo $((a ** b))  # Potencia → 1000

# Incremento / Decremento
((a++))          # a = 11
((a+=5))         # a = 16
((b*=2))         # b = 6

# Decimales con bc
resultado=$(echo "scale=2; 10/3" | bc)
echo "10/3 = $resultado"  # → 3.33

Leer archivos línea a línea

bash · leer_archivo.sh 
#!/bin/bash

# Método 1: while + read
while IFS= read -r linea; do
  echo "→ $linea"
done < "archivo.txt"

# Método 2: con cat y pipe
cat archivo.txt | while read linea; do
  echo "$linea"
done

# Leer CSV: separar por coma
while IFS="," read -r col1 col2 col3; do
  echo "$col1 | $col2 | $col3"
done < "datos.csv"
IFS (Internal Field Separator) controla cómo Bash divide las palabras. IFS= (vacío) preserva espacios y tabulaciones en cada línea. -r evita que las barras invertidas se interpolen.

Operadores && y ||

#!/bin/bash

Bucles

Bucle for

bash · for.sh 
#!/bin/bash

# Iterar sobre una lista
for fruta in manzana pera naranja uva; do
  echo "Fruta: $fruta"
done

# Iterar con rango de números
for i in {1..5}; do
  echo "Número: $i"
done

# Rango con paso: {inicio..fin..paso}
for i in {0..20..5}; do
  echo -n "$i "     # → 0 5 10 15 20
done

# Estilo C
for ((i=0; i<5; i++)); do
  echo "i = $i"
done

# Iterar sobre archivos
for archivo in *.txt; do
  echo "Procesando: $archivo"
done

# Iterar sobre la salida de un comando
for usuario in $(cut -d: -f1 /etc/passwd); do
  echo $usuario
done

Bucles while y until

bash · while_until.sh 
#!/bin/bash

# while — se ejecuta MIENTRAS la condición sea verdadera
contador=1
while [[ $contador -le 5 ]]; do
  echo "Vuelta $contador"
  ((contador++))
done

# until — se ejecuta HASTA que la condición sea verdadera
n=1
until [[ $n -gt 5 ]]; do
  echo "n = $n"
  ((n++))
done

# Bucle infinito con break
while true; do
  read -p "¿Continuar? (s/n): " resp
  if [[ $resp == "n" ]]; then
    break
  fi
  echo "Continuando..."
done

# continue — salta a la siguiente iteración
for i in {1..10}; do
  [[ $((i % 2)) -eq 0 ]] && continue   # saltar pares
  echo -n "$i "   # → 1 3 5 7 9
done

Funciones

Crear funciones reutilizables, pasar parámetros y devolver valores.

Definición y llamada

bash · funciones.sh 
#!/bin/bash

# Dos formas de definir funciones
function saludar {
  echo "¡Hola desde la función!"
}

despedir() {
  echo "¡Hasta pronto!"
}

# Llamar la función (sin paréntesis)
saludar
despedir

# Función con parámetros ($1, $2...)
function sumar {
  local resultado=$(($1 + $2))
  echo "$resultado"   # "devolver" el valor con echo
}

total=$(sumar 4 7)
echo "4 + 7 = $total"   # → 11

# Variables locales con local
function test_scope {
  local interna="solo dentro"
  global="visible fuera"
}

test_scope
echo $global     # → visible fuera
echo $interna    # → (vacío, no existe fuera)

# Código de retorno con return
function es_par {
  [[ $(($1 % 2)) -eq 0 ]] && return 0 || return 1
}

es_par 4 && echo "4 es par"
Local vs Global: Usa siempre local para las variables dentro de funciones. Sin local, las variables son globales y pueden causar efectos secundarios inesperados.

Arrays indexados

#!/bin/bash

# Declarar array
frutas=("manzana" "pera" "naranja" "uva")

# Acceder por índice (empieza en 0)
echo ${frutas[0]}      # → manzana
echo ${frutas[2]}      # → naranja
echo ${frutas[-1]}     # → uva (último)

# Todos los elementos
echo ${frutas[@]}      # → manzana pera naranja uva

# Número de elementos
echo ${#frutas[@]}     # → 4

# Añadir elemento
frutas+=("mango")

# Modificar elemento
frutas[1]="kiwi"

# Eliminar elemento
unset frutas[3]

# Iterar sobre el array
for fruta in ${frutas[@]}; do
  echo "- $fruta"
done

# Iterar con índice
for i in ${!frutas[@]}; do
  echo "[$i] ${frutas[$i]}"
done

Procesos y señales

Gestión de procesos, ejecución en segundo plano y captura de señales del sistema.

Procesos en segundo plano

bash · procesos.sh 
#!/bin/bash

# Ejecutar en segundo plano con &
sleep 10 &
PID_SLEEP=$!
echo "Sleep PID: $PID_SLEEP"

# Esperar a que un proceso termine
wait $PID_SLEEP
echo "Sleep terminó"

# Lanzar múltiples procesos y esperar todos
for i in {1..3}; do
  sleep $i &
done
wait   # Espera todos los jobs en segundo plano
echo "Todos terminaron"

# Matar un proceso
kill $PID_SLEEP       # SIGTERM (cortés)
kill -9 $PID_SLEEP    # SIGKILL (forzado)

Captura de señales con trap

bash · trap.sh 
#!/bin/bash

# Función de limpieza
function cleanup {
  echo ""
  echo "Script interrumpido. Limpiando..."
  rm -f /tmp/mi_script_temp_*
  exit 1
}

# Capturar Ctrl+C (SIGINT) y SIGTERM
trap cleanup INT TERM

# Ejecutar al salir (siempre, haya error o no)
trap 'echo "Script finalizado"' EXIT

# Capturar errores con ERR
trap 'echo "Error en línea $LINENO"' ERR

echo "Ejecutando tarea larga..."
sleep 30    # Ctrl+C aquí llamará cleanup()
Buena práctica: Siempre usa trap cleanup EXIT en scripts que crean archivos temporales. Así se limpian aunque el script falle o sea interrumpido.

Expresiones regulares

Búsqueda y manipulación de texto con grep, sed y awk.

grep — búsqueda de patrones

bash · grep.sh 
#!/bin/bash

# Búsqueda básica
grep "error" app.log            # líneas con "error"
grep -i "error" app.log         # sin distinción mayúsc.
grep -n "error" app.log         # con número de línea
grep -v "error" app.log         # líneas SIN "error"
grep -c "error" app.log         # contar coincidencias
grep -r "TODO" ./src/           # recursivo en carpeta

# Expresiones regulares extendidas con -E o egrep
grep -E "error|warning" log.txt  # error O warning
grep -E "^[0-9]{4}-" log.txt     # líneas que empiezan con 4 dígitos y guión
grep -E "[a-z]+@[a-z]+\.[a-z]+" emails.txt  # emails

# En Bash, [[ ]] soporta regex con =~
email="usuario@ejemplo.com"
if [[ $email =~ ^[a-zA-Z0-9]+@[a-zA-Z0-9]+\.[a-zA-Z]+$ ]]; then
  echo "Email válido"
fi

#!/bin/bash

# Sustituir primera ocurrencia por línea
sed 's/foo/bar/' archivo.txt

# Sustituir todas las ocurrencias (g = global)
sed 's/foo/bar/g' archivo.txt

# Edición in-place (modifica el archivo)
sed -i 's/VERSION=.*/VERSION=2.0/g' config.env

# Eliminar líneas que coinciden
sed '/^#/d' config.txt       # borrar comentarios
sed '/^$/d' archivo.txt      # borrar líneas vacías

# Mostrar solo líneas 5 a 10
sed -n '5,10p' archivo.txt

sed '3i\Nueva línea antes de 3' archivo.txt
sed '3a\Nueva línea después de 3' archivo.txt

Buenas prácticas

Siempre usar

set -euo pipefail al inicio de todo script de producción. Evita errores silenciosos.

Variables entre comillas

Siempre entrecomilla variables: "$var" en lugar de $var. Previene problemas con espacios.

Validar argumentos

Comprueba que los argumentos necesarios existen y son del tipo correcto antes de usarlos.

ShellCheck

Usa shellcheck script.sh — analizador estático gratuito que detecta errores comunes.

#!/usr/bin/env bash
# Descripción: Plantilla robusta para scripts de producción
# Uso: ./script.sh [opciones] argumento

set -euo pipefail
IFS=$'\n\t'

# Variables globales

readonly SCRIPT_NAME=$(basename "$0")

}

function error {
  log "ERROR: $*"
  exit 1
}

  log "Iniciando con argumento: $1"
  # ... tu código aquí
}
